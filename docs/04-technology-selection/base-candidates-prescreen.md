# AIDevOps 底座选型 — 预筛报告

> 基于 11 个候选项目的轻量级预筛选，目的是淘汰明显不适合的项目，确定进入深度分析的短名单。

## 1. 评估框架

### 核心考察维度（决定选不选）

| 维度 | 权重 | 评估要点 |
|------|------|----------|
| **① Agent 核心能力** | 🔴 最高 | 对话式 Agent 是否成熟、多轮对话、上下文管理、流式输出 |
| **② 可扩展性** | 🔴 最高 | Hook 机制、Skill 加载、MCP 支持、工具注册模式 |
| **③ 架构质量** | 🔴 最高 | 代码分层、模块化程度、Fork 后可理解性 |

### 重要参考维度（加分项）

| 维度 | 权重 | 评估要点 |
|------|------|----------|
| **④ Web 对话前端** | 🟡 高 | 是否自带 Web 端、对话 UI 质量 |
| **⑤ 改造量估算** | 🟡 高 | 达到穿刺版效果需要做多少改造 |
| **⑥ 社区/稳定性** | 🟡 高 | Star、提交频率、License |

### 低优先级（我们自己建）

多租户/RBAC、工作流引擎、DevOps 集成、通知/审计、产物/确认机制

## 2. pi 项目分析

| 项 | 值 |
|---|---|
| 路径 | `/Users/hzz/workspace/pi/` |
| 定位 | 终端 AI 编码 Agent 框架（类似 OpenCode） |
| 技术栈 | TypeScript / Node.js monorepo |
| 版本 | v0.75.3 |
| License | MIT |

### 架构

```
pi/
├── packages/
│   ├── ai/              # 统一 LLM API 层（20+ Provider）
│   ├── agent/           # Agent 运行时核心（stateful agent loop）
│   ├── coding-agent/    # CLI 编码 Agent 应用
│   └── tui/             # 终端 UI 框架
```

### 能力评估

| 能力 | 状态 | 说明 |
|------|------|------|
| Agent 核心 | ✅ 成熟 | stateful agent loop + 事件流 |
| Skill 系统 | ✅ 完整 | 符合 agentskills.io 标准 |
| MCP | ❌ 故意不支持 | 需自建扩展 |
| Web 前端 | ❌ 纯终端 | web-ui 包为空壳 |
| 扩展机制 | ✅ 事件驱动 | 可注册工具/命令/拦截器 |
| LLM 支持 | ✅ 20+ | OpenAI/Claude/Gemini/Bedrock/DeepSeek 等 |

### 结论

pi 的 `agent-core` 和 `ai` 包质量很高，但它本质是 CLI Agent，不是平台。**可以作为 Agent 引擎嵌入，但不适合做整个平台的底座。**

## 5. 淘汰决策

### 直接淘汰（5 个）

| 项目 | 淘汰原因 |
|------|----------|
| **AutoGen** | 维护模式 + CC-BY-4.0 限制商用 |
| **MetaGPT** | 停更 + 无 Web UI + 架构局限（角色 SOP 模式太特殊） |
| **LangGraph** | 纯基础设施层，不是可 Fork 的平台/框架，需要自己搭一切 |
| **Google ADK** | 太新（v2.0 刚出）+ Google 生态绑定风险 |
| **OpenAI Agents SDK** | 太轻量，纯 SDK 无平台形态，Fork 价值低 |

### 进入深度分析（5 个）

| 项目 | 入围理由 | 核心优势 | 核心风险 |
|------|----------|----------|----------|
| **Dify** | 功能最全的平台，总分最高 | Web UI 成熟、企业功能齐全、MCP 支持、工具生态 | License 有附加条款、体量大定制成本高 |
| **Langflow** | Stars 最高、MIT License | 可视化编排、MCP 支持、Python 生态活跃 | 体量大、架构可控性待验证 |
| **CrewAI** | 独立轻量框架 | MIT、无 LangChain 依赖、Crews+Flows 双模型 | 无 Web UI，需自建前端 |
| **Mastra** | TS 原生、架构最现代 | MCP、Hook 机制、40+ LLM、模块化设计 | 双 License（ee/ 目录受限）、社区较小 |
| **Flowise** | TS 全栈可视化平台 | Apache 2.0、低代码、自带 Web UI | 贡献者仅 124 人、深度定制能力待验证 |
