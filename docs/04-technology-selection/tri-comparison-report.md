# AIDevOps 底座平台三选一技术评估报告

> **报告日期**：2026-05-21
> **评估对象**：pi v0.75.3 / Dify / Mastra
> **报告用途**：技术选型决策参考

---

## 一、执行摘要

**结论**：推荐以 **Mastra** 作为 AIDevOps 平台的核心底座。

**核心依据**：

| 维度 | pi | Dify | Mastra |
|------|:---:|:---:|:------:|
| 综合加权得分 | 2.15 | **3.35** | 3.30 |
| 推荐优先级 | ❌ 不推荐 | ✅ 首选 | ⚠️ 备选 |

**推荐理由**：
1. **平台完整度最高**：Dify 开箱即用——Web UI、多租户（5 级 RBAC）、MCP、803 插件生态、工作流引擎（21 种节点），AIDevOps 60% 的功能已直接具备
2. **改造量最小**：裁剪+加业务页面+加 Skill 注入，总改造量约 3-4 个月，远低于 Mastra 的从零建平台（4+ 个月）
3. **社区生态最强**：141K stars、40+ 财富 500 客户、持续迭代有保障
4. **改造方向务实**：在成熟平台上做加法（增强 Agent 核心、加 Skill），比从零建平台风险更低

**风险提示**：
- Dify Agent 核心较薄（2.3K LOC），无 SubAgent/Hook，需扩展增强
- 代码量大（460K LOC），裁剪需谨慎解耦 RAG 耦合

## 四、逐维评估

### D1：Agent 对话核心能力（权重 20%）

#### pi
- **得分**：4/5
- **改造难度**：Easy
- **改造量**：0 人天
- **论证**：pi 的 agent-core 采用状态化事件驱动循环，架构清晰，JSONL 会话管理完善。作为对话引擎质量极高，可直接作为 AIDevOps 的核心对话模块。

#### Dify
- **得分**：3/5
- **改造难度**：Medium
- **改造量**：20-30 人天
- **论证**：Agent 核心代码 2,301 LOC，FC + CoT 两种策略能覆盖基础对话场景。7 种 AppMode（CHAT/AGENT_CHAT/WORKFLOW 等）提供了丰富对话模式。graphon DAG 引擎（21 种节点）支持复杂编排。短板在于 Agent 核心无 Hook/Skill 注入/SubAgent，需扩展约 2-3K LOC 增强。改造量中等而非"巨大"——框架已在运行，是在已有基础上增强。

#### Mastra
- **得分**：5/5
- **改造难度**：Easy
- **改造量**：0 人天
- **论证**：Agent 核心达 7,086 行代码，具备完整的 StreamProcessor、MessageList 机制。对话状态管理、消息历史、多轮推理均为原生实现。7 种 AppMode 支持不同场景。

**D1 维度的加权得分对比**：pi=0.80, Dify=0.60, Mastra=**1.00**

### D2：Skill/知识动态注入（权重 15%）

#### pi
- **得分**：5/5
- **改造难度**：Easy
- **改造量**：0 人天
- **论证**：pi 完整支持 agentskills.io 标准，Markdown-based 动态加载，开箱即用。

#### Dify
- **得分**：2/5
- **改造难度**：Medium
- **改造量**：4-5 人天
- **论证**：Dify 无 Skill 动态加载机制，但有丰富的插件生态（803 个 Marketplace 插件）和工具注册系统（5 种 Provider）。插件机制本身是静态扩展，但为 Skill 系统提供了良好基础——只需在 `_organize_prompt_messages()` 层增加动态匹配+注入逻辑，4-5 人天可实现。底层能力比"几乎无能力"要好。

#### Mastra

- **改造难度**：Easy
- **改造量**：1-2 人天
- **论证**：Mastra 通过 Hook 系统 + 动态 Prompt 组装实现 Skill 动态注入，机制自然契合。预计 1-2 人天完成 Skill 标准化封装。

## 八、最终推荐

### 8.1 推荐方案：**Dify 渐进式改造**

**核心策略**：以 Dify 作为完整平台底座，裁剪不需要的模块，增强 Agent 核心能力，补充 Skill 动态注入和外部 Agent 调度。

**执行路径**：

```
阶段 1（月 1）：裁剪瘦身
  - 裁向量库（Weaviate→pgvector）、裁无关 AppMode
  - 12 容器 → 7 容器，460K → ~150K LOC

阶段 2（月 2）：业务定制 + Skill 增强
  - 加 AgentCenter 业务页面（看板/概览/上下文设置）
  - 在 _organize_prompt_messages() 层加 Skill 动态注入（4-5 人天）

阶段 3（月 3）：Agent 核心增强
  - 扩展 core/agent/，增加 Hook 机制
  - 设计 SubAgent 接口，对接 ACP 协议
  - Agent 核心从 2.3K LOC → ~5K LOC

阶段 4（月 4）：生产加固
  - 安全审计（沙箱 per-tenant 隔离）
  - 审计追溯增强
  - 监控告警
```

### 8.2 备选方案：Mastra 从零建平台

如团队更看重 Agent 核心质量，且愿意投入更多时间构建平台：
- 优点：Agent 核心成熟（7K LOC）、SubAgent 原生支持、Hook 系统、TypeScript 全栈
- 缺点：需从零构建 Web 平台（60+ 人天）和多租户（10-15 人天）
- 策略：4 个月出可用版本，长期架构质量更优

### 8.3 不推荐方案：pi

pi 不适合作为 AIDevOps 平台底座，原因：
1. 无 MCP 支持，与工具生态需求根本矛盾
2. 无 Web UI，与平台化需求根本矛盾
3. 单用户架构，多租户改造代价最高
4. 建议将 pi 作为**嵌入式对话引擎**模块集成，而非平台底座
