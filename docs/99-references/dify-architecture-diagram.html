📡 典型请求流：用户发送消息
1
用户在浏览器输入消息 → POST /api/chat/messages
2
Nginx 路由到 API Server (Flask) → 认证 + 多租户隔离 (tenant_id)
3
根据 App 类型分发到对应执行引擎 → Agent Engine / Chat Engine / Workflow Engine
4
Agent Engine 决策 → 是否需要调用工具？ → Tool System / MCP Client
5
构建 Prompt → 通过 Plugin Daemon 调用智谱 GLM API → SSE 流式返回
6
流式 chunks → Nginx → 浏览器 → 用户看到打字效果 ✅
7
对话完成 → Celery Worker 异步持久化 → PostgreSQL 存储消息记录
