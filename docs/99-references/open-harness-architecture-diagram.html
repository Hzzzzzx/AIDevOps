🔍 AIDevOps 差距分析
Open Harness 现有能力 → AIDevOps 所需能力
✅ 核心优势（可直接复用）
• 事件溯源：完整审计链，所有操作可追溯
• VCR 录制：零成本回放，测试/开发利器
• 时间旅行：任意状态回溯，调试神技
• 会话分叉：探索多策略，不破坏主线
• Effect 生态：类型安全 + 依赖注入 + 组合
• Zod 校验：结构化输出保证
• HITL：人机协作审批机制
• TypeScript：前后端统一类型
❌ 关键缺失（需扩展）
• MCP 支持：无法接入外部工具协议
• Hook 系统：无生命周期拦截器
• Skill 系统：无技能注册/发现机制
• SubAgent：无多代理协作
• 多 Provider：仅 Anthropic，无法用 GLM 等
• 多租户：无租户隔离
• Auth/RBAC：无认证权限
• Web UI：无可视化界面
🏗️ 架构原则
A
Effect 不可泄漏：Public API 只暴露 Promise，Effect 完全封装在内部
B
事件即真相：所有状态变更通过事件记录，State = fold(Events)
C
三层分离：Domain（纯类型） → Services（依赖注入） → Programs（副作用）
D
VCR 模式：录制 Provider 调用 → 存储到 LibSQL → 回放时零 API 成本
E
分叉即复制：Clone 事件流 → 独立分支 → 不影响原会话
